Python as an object-oriented programming language

A class is a blueprint or template for creating objects. It defines the attributes (properties) and methods (functions) that its objects will have. For example:

In[2]:
class Mammal:
    def __init__(self, name, breed):
        self.name = name     # attribute
        self.breed = breed   # attribute

    def sound(self):          # method
        print(f"{self.name} makes a sound!")

Here, Mammal is a class that specifies what a "mammal" object should contain (name, breed), and what it can do (make a sound).

A method is a function defined inside a class that operates on objects of that class. The first parameter of a method is always self, which represents the instance calling the method.

A concrete realisation of a class is an object. For example:

In[3]:
my_cat = Mammal('Max', 'Maine Coon')
print(my_cat.name)
my_cat.sound() 

Max
Max makes a sound!

Here, my_cat is a specific instance of a class Mammal. It's a Main Coon called Max.

A class can inherit the attributes and methods of another class. This is useful when you need to work with subcategories. For example, cats are mammals, so they can do all the same things that other mammals do, and more:

In[4]:
class Mammal:
    def __init__(self, name, breed):
        self.name = name     # attribute
        self.breed = breed   # attribute

    def sound(self):          # method
        print(f"{self.name} makes a sound!")

class Cat(Mammal):

    def meow(self):
        print(f"{self.name} says meow!")
In[5]:
c = Cat('Whiskers', 'British Shorthair')

print(c.name)
print(c.breed)
c.meow()

Whiskers
British Shorthair
Whiskers says meow!

We can also redefine Mammal to allow custom sounds:

In[8]:
class Mammal:
    def __init__(self, name, breed, sound):
        self.name = name
        self.breed = breed
        self._sound = sound

    def sound(self):
        print(f"{self.name} says {self._sound}")
In[9]:
dog = Mammal('Bluey', 'Australian Shepherd', 'woof')
cat = Mammal('Whiskers', 'British Shorthair', 'meow')

dog.sound() 
cat.sound()

Bluey says woof
Whiskers says meow

Classes can be very useful in research. Suppose you are doing a study on how teachers adapt their teaching methods to different classroom contexts. You conduct interviews, collect observation notes, and analyse reflective journals. You want to organise, analyse, and retrieve your data efficiently. Each participant (teacher) has multiple attributes (e.g., name, years of experience, subjects taught, coded themes) and associated qualitative data (e.g., interview transcripts, observation notes). You can create a Teacher class to model each participant as an object with attributes and methods. This would make it easier to store, retrieve, and manipulate data programmatically instead of juggling each teacher's data separately.

In[10]:
class Teacher:
    def __init__(self, name, experience, subjects):
        self.name = name
        self.experience = experience
        self.subjects = subjects
        self.interviews = [] # list of interviews
        self.observations = [] # list of notes
        self.codes = [] # list of codes

    def add_interview(self, transcript):
        self.interviews.append(transcript)

    def add_observation(self, note):
        self.observations.append(note)

    def add_code(self, code):
        self.codes.append(code)

    def summary(self):
        print(f"Teacher: {self.name}")
        print(f"Experience: {self.experience} years")
        print(f"Subjects: {', '.join(self.subjects)}")
        print(f"Number of Interviews: {len(self.interviews)}")
        print(f"Codes: {', '.join(self.codes)}")

Lists

Lists are created using square brackets, like so:

In[29]:
colours = ['blue', 'red', 'green', 'yellow', 'purple']
print(colours)

['blue', 'red', 'green', 'yellow', 'purple']

Lists can contain any kind of data, including a mix of different data types. For example:

In[31]:
mixed_list = [1, 'Bluey', 3.679, False]

How can we access the second item in this list? In R, we would do an equivalent of:

In[32]:
print(mixed_list[2])

3.679

But here, we get the third element, not the second. This is because, in Python, counting starts from 0, not from 1.

In[33]:
print(mixed_list[1])

Bluey

We can also extract a range of items from a list; this is called slicing:

In[34]:
print(mixed_list[1:3])

['Bluey', 3.679]

Take a look at this output. Is this what you expected to happen?

In Python, a range of numbers is specified in the inclusive:exclusive format. This means that the number before the colon (:) is included in the range, but the number after the colon is not included in the range. So, when we use print(mixed_list[1:3]), we are asking Python to extract item 1 (i.e., the second item in the list) up to, but not including, item 3. To extract items 1 to 3, including item 3, we need to write:

In[35]:
print(mixed_list[1:4])

['Bluey', 3.679, False]

Lists can also contain other lists:

In[36]:
my_pets = [ ['cat', 'Whiskers'], ['dog', 'Bluey'] ]
print(my_pets[0])
print(my_pets[0][1])

['cat', 'Whiskers']
Whiskers

Another important thing about lists is that you do not need to create a for-loop to generate lists. Instead, you can use a method called list comprehension this is an example of a feature that is very Pythonic.

For example, to create a list of the squares of the numbers from 1 to 10, we could use a for-loop:

In[37]:
squares = []
for x in range(1, 11):
    squares.append(x**2)
print(squares)

[1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

However, list comprehension would be much shorter and also more elegant:

In[38]:
squares = [x**2 for x in range(1, 11)]
print(squares)

[1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

You can even combine list comprehension with a conditional statement. For example, let's make a list of squares for odd numbers only:

In[39]:
squares = [x**2 for x in range(1, 11) if x % 2 != 0]
print(squares)

[1, 9, 25, 49, 81]

Now let's combine what we've learned about objects and lists:

In[40]:
class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade

students = [Student('Alice', 90), Student('Bob', 85), Student('Charlie', 92)]

high_scorers = [s.name for s in students if s.grade > 88]
print(high_scorers)

['Alice', 'Charlie']

We created a Student class with the attributes name and grade, then made a list containing three objects of this class. Using list comprehension, we then generated a new list with the names of those students whose grade is above 88. Neat!

Dictionaries

A dictionary is a collection of key-value pairs. Unlike lists, where you access items by their numeric index, dictionaries let you retrieve values using their associated keys.

In[48]:
my_pets = {'cat':'Whiskers', 'dog':'Bluey'}
cat_name = my_pets['cat']
print(f'My cat is called {cat_name}.')

My cat is called Whiskers.

This example also uses an f-string (formatted string). F-strings allow you to insert variables directly into strings using braces {}.

Updating a dictionary is very easy. For example, we can modify the cat's name in the dictionary my_pets like so:

In[49]:
my_pets['cat'] = 'Max'
print(my_pets)

{'cat': 'Max', 'dog': 'Bluey'}

We can also add a new pet to the list:

In[50]:
my_pets['rat'] = 'Nibbles'
print(my_pets)

{'cat': 'Max', 'dog': 'Bluey', 'rat': 'Nibbles'}

Or, if you want to add several rats:

In[52]:
del my_pets['rat']
my_pets['rats'] = ['Nibbles', 'Oats', 'Squeak']
print(my_pets)

{'cat': 'Max', 'dog': 'Bluey', 'rats': ['Nibbles', 'Oats', 'Squeak']}

We can do this because, in Python, both lists and dictionaries are mutable.

We can change students from the example above into a dictionary by using student names as keys and Student objects as values:

In[53]:
class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade

students = {
    'Alice': Student('Alice', 90),
    'Bob': Student('Bob', 85),
    'Charlie': Student('Charlie', 92)
}

high_scorers = [s.name for s in students.values() if s.grade > 88]
print(high_scorers)

['Alice', 'Charlie']

Instead of creating a list with the names of students who achieved high scores, we can create a dictionary that includes both their names and scores:

In[54]:
high_scorers = {name: s.grade for name, s in students.items() if s.grade > 88}
print(high_scorers)

{'Alice': 90, 'Charlie': 92}

Can you read this code easily? This example illustrates that although list and dictionary comprehensions are powerful, they can quickly become difficult to read, especially for less experienced programmers.

Regular expressions

In[15]:
import re # imports the module needed to work with RegEx in Python

text = 'The winning numbers are 123, 456, and 789.'
numbers = re.findall(r"\d+", text)
print(numbers)

['123', '456', '789']

Here, we are trying to find all numbers used in a string text. The expression \d+ means 'one or more digits', and the method .findall() returns all matches as a list.

Suppose you need to extract all email addresses from a text. We can achieve this by using regular expressions in a simple function:

In[16]:
import re

def extract_emails(text):
    """Extracts all email addresses from a string."""
    pattern = r'[\w\.-]+@[\w\.-]+\.\w+'
    return re.findall(pattern, text)

How does this work? The pattern looks for all occurrences where a sequence of one or more letters, numbers, underscores, dots, or hyphens ([\w\.-]+) is followed by the literal @ symbol. After the @, it matches another sequence of one or more letters, numbers, underscores, dots, or hyphens ([\w\.-]+), followed by a dot and the top-level domain (one or more word characters) (\.\w+). Let's check if that works:

In[17]:
text = 'Contact us at support@example.com, sales@example.org, or admin@my-site.co.uk.'
emails = extract_emails(text)
print(emails)

['support@example.com', 'sales@example.org', 'admin@my-site.co.uk']

Great!

Now let's calculate some basic descriptive statistics from our DataFrame. For example, we can find the average score for students who passed the test and for those who didn't:

Name: Score, dtype: float64

Here, we are using the .groupby method to split the DataFrame into groups based on whether the students passed the test and then apply the .mean() operation to each group.

You can apply lots of simple operations in a similar manner (e.g., .min(), .max(), .sum(), .median(), .count(), .std()), but for more complex operations (e.g., mathematical functions like log or exp, random number generation, advanced linear algebra) you will need to import a package called NumPy (and see this tutorial for a list of other useful packages).

In this example, we already had subject information for Alice and Bob stored in a separate DataFrame, and we used the .merge method to add these extra details to our original DataFrame. We told Pandas to merge based on the Name column, and we set how = left to keep all the rows from the left DataFrame (the one we started with) and bring in matching data from the right DataFrame where possible. Now let's compute the mean and one standard error of the mean for the variable YearsExperience:

In[55]:
mean_exp = df['YearsExperience'].mean()
se_exp = df['YearsExperience'].std(ddof = 1) / np.sqrt(len(df))

Here, I set ddof = 1 to specify the degrees of freedom. This makes the denominator in the standard deviation formula n - 1, which means we're calculating the sample standard deviation rather than the population standard deviation. In this context, the distinction isn't important, so it's fine to ignore it if it's unfamiliar.
